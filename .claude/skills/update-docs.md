# Update Docs Skill

You are maintaining the documentation system for the c64u-bbs repository (github.com/brainvat/c64u-bbs).

## Step 1: Re-index docs/toc.json

Run the Python indexer to scan all .md and .html files in docs/ and rebuild the TOC index:

```bash
python3 scripts/update_docs_toc.py
```

Read the updated `docs/toc.json` to understand what files exist.

## Step 2: Regenerate docs/TOC.md

Read every file listed in `docs/toc.json`. Then write a new `docs/TOC.md` that:

- Starts with `# Documentation — Table of Contents`
- Has a note: `> Auto-generated by the update-docs skill. Do not edit manually.`
- Organizes the documents into logical sections with clear headings
- For each document, provides the filename as a relative link and a helpful 1-2 sentence description based on the file's actual content (not just the title)
- Uses a clean markdown table or organized list format
- Excludes TOC.md itself from the listing (it's self-referential)

## Step 3: Update docs/index.html

Read the current `docs/index.html`. Update it to reflect the current state of the project:

- Keep the Commodore 64 retro aesthetic (blue background, monospace font, C64 color palette)
- Update the Features section to accurately list implemented features based on what exists in the codebase
- Update the Documentation section to link to all docs listed in toc.json
- Keep the structure and styling intact; only update content sections
- If major new features have been added to the codebase, add them to the features list

## Step 4: Update README.md

Read the current root `README.md`. Update it to contain:

- Project name and one-line description
- Essential installation/setup requirements (Python version, dependencies, how to install hooks)
- Quick-start instructions
- A link to the full documentation: `https://brainvat.github.io/c64u-bbs/`
- A link to the GitHub repository: `https://github.com/brainvat/c64u-bbs`
- Keep it concise — README is for essential info only, not full documentation

## Step 5: Re-run the indexer

After updating TOC.md, index.html, and README.md, re-run the indexer so toc.json reflects the updated file sizes and timestamps:

```bash
python3 scripts/update_docs_toc.py
```

## Step 6: Stage and verify

Stage all changed docs files and verify the pre-push hook would pass:

```bash
git add docs/toc.json docs/TOC.md docs/index.html README.md
```

Tell the user the docs have been updated and are ready to commit.
