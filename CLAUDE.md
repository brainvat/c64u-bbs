# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

c64u-bbs — a Commodore 64 Unicode BBS. Repository: github.com/brainvat/c64u-bbs (PUBLIC).

## Security — CRITICAL

This is a **public repository**. Every commit is visible to everyone.

- Git hooks in `.githooks/` scan for secrets (passwords, API keys, tokens, private keys, connection strings) at both commit and push time.
- Hooks must be installed: `bash scripts/install-hooks.sh` or `git config core.hooksPath .githooks`
- Never commit `.env`, credentials, tokens, or private keys — even temporarily.
- Never use `--no-verify` to bypass hooks without explicit user approval.
- The `.gitignore` blocks common sensitive file patterns, but hooks provide the real enforcement.

## Docs System

The `docs/` directory is auto-managed:

- **`docs/toc.json`** — Machine-readable index of all `.md` and `.html` files in `docs/`. Generated by `scripts/update_docs_toc.py`.
- **`docs/TOC.md`** — Human-readable table of contents. Written by the `/update-docs` skill using LLM capabilities.
- **`docs/index.html`** — Project landing page (GitHub Pages). Retro C64 aesthetic.

**Pre-push hook** blocks pushes if docs files are out of sync with `toc.json`. When this happens:

1. Run `python3 scripts/update_docs_toc.py` to re-index, OR
2. Use the `/update-docs` skill which handles everything: re-indexes, rewrites TOC.md, updates index.html, and updates README.md.

## Journal System

`docs/journal/` contains development journal entries, managed separately from the docs TOC.

- **`docs/journal/index.html`** — Journal index page listing all entries (newest first).
- **`docs/journal/{entry-name}/index.html`** — Individual journal entries.
- The TOC indexer and pre-push hook **intentionally ignore** `docs/journal/`. Journals are not tracked in `toc.json`.
- Use `/journal` to create a new entry. This is manual/on-demand — unlike `/update-docs` which runs automatically.

## Key Commands

```bash
# Install git hooks
bash scripts/install-hooks.sh

# Re-index docs (updates toc.json only)
python3 scripts/update_docs_toc.py

# Full docs update (use the Claude Code skill)
/update-docs

# Create a journal entry (on-demand, user-initiated)
/journal
```

## Debugging Rule: Hold the Phone

When a workaround fails twice, **stop**. Do not attempt a third workaround. Instead:

1. State the problem simply to the user
2. Ask: "Does this even make sense? Can we prove it with a simple test?"
3. **Ask the human for an idea** — they may solve it in 30 seconds with a Google search
4. Apply Occam's razor before inventing elaborate theories

Never accept absurd conclusions ("the drive is read-only", "the ACIA is broken") without challenging them first.
